## 3. 技术实现方案

- 行情服务：模拟行情（随机游走算法），接口预留供AKShare替换
- WebSocket：基于 ws 库，增量推送（仅推送 bid/ask 变化的品种）
- K线：实时聚合到内存，定时持久化到 MySQL
- 预警：轮询扫描（5秒间隔），触发后更新状态+执行通知
- K线图表：UniApp Canvas 2D API 直接绘制

## 4. 技术约束与集成方案

- 全部代码集成在「服务端/」主后端体系
- 接口路径统一为 /api/admin 和 /api/mobile
- 响应格式统一为 { code, msg, data }
- 管理接口统一使用 authenticateAdmin 鉴权
- 用户接口统一使用 authenticateToken 鉴权

## 5. 任务边界限制

- 本阶段不实现交易下单逻辑（品种详情页"买入做多/卖出做空"按钮仅做占位提示）
- 模拟盘费用配置在阶段六实现
- 行情数据当前为模拟数据，正式上线前需替换为真实数据源
