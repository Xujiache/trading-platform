## 3. 技术实现方案

### 3.1 系统监控
- `GET /api/admin/system/health`：综合检查DB连接+延迟、Redis PING+延迟、连接池状态、行情源健康、系统资源
- `GET /api/admin/health`：轻量检查，返回 healthy/degraded/unhealthy
- `GET /api/mobile/health`：移动端增强版，检测DB/Redis/行情源
- `GET /api/admin/monitor/connections`：DB连接池（total/idle/active/waiting）+ Redis（clients/memory）

### 3.2 CSV 数据导出
- `GET /api/admin/export/:type`：流式生成CSV，UTF-8 BOM，最大50000条
- 支持 `start_date`/`end_date`/`status`/`user_id` 筛选参数
- 前端 HTTP 工具已适配 `responseType: 'blob'` 绕过 JSON 拦截器

### 3.3 批量处理
- 每种批量操作上限 100 条，逐条事务执行，返回 success/failed 明细
- 入金确认：检查 pending 状态→更新→创建资金账户→写入资金流水
- 出金审核：支持 approve/reject/complete，驳回自动退回余额
- 用户冻结/解冻：状态检查后批量更新

### 3.4 审计日志
- 独立 `audit_logs` 表，与 `operation_logs` 分离
- `auditLog` 中间件：拦截 `res.json`，自动记录操作人/模块/操作/目标/IP/状态/耗时
- `writeAuditLog` 函数：支持直接写入审计记录（含变更前后数据）
- 已挂载路由：fund（8个接口）、risk（3个接口）、trade（3个接口）、users（3个接口）

### 3.5 稳定性增强
- 优雅停机：关闭HTTP→停行情监控→关DB连接池→关Redis，15s超时强制退出
- 全局异常：uncaughtException带时间戳+堆栈，EADDRINUSE/EACCES自动退出
- 行情监控：增加恢复历史（最近50条）、60s冷却期、健康后自动重置计数器

## 4. 技术约束与集成方案

- CSV 导出：大数据量通过 LIMIT 50000 控制，支持分页导出
- 批量操作：逐条事务保证原子性，失败不影响其他记录
- 审计日志：异步写入，不阻塞请求响应；与现有操作日志体系并行
- HTTP 工具：blob/arraybuffer 响应类型绕过 JSON 拦截器直接返回原始数据
