# CONSENSUS - 阶段三：交易引擎、订单、费用、账户体系

## 1. 明确的需求描述

本阶段实现交易系统核心引擎，包括市价单/挂单/平仓/止损止盈/移动止损全流程，费用计算引擎（6种费用），4个定时监控任务，Redis并发控制，以及用户端和后台管理界面。

## 3. 技术实现方案

- 交易引擎: `services/tradeEngine.js` — 核心下单/平仓/挂单逻辑，事务保证原子性
- 费用计算: `services/feeCalculator.js` — 保证金/点差/手续费/隔夜费/盈亏计算
- 账户服务: `services/accountService.js` — 资金冻结/释放/结算/流水记录
- 分布式锁: `utils/distributedLock.js` — Redis SET NX + Lua释放，按用户维度加锁
- 定时任务: 4个独立服务模块，在app.js启动时初始化

## 4. 技术约束与集成方案

- 交易操作必须使用 Redis 分布式锁
- 费用计算必须与品种配置参数对齐
- 行情数据来自阶段二的 WebSocket 服务

## 5. 任务边界限制

- 本阶段不实现资金入金/出金
- 风控强平在阶段五实现，本阶段仅预留接口
- 前端下单页隐藏点差成本展示，但后端必须计算
