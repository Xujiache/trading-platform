### 3. 技术决策记录

| 决策项 | 选择 | 原因 |
|--------|------|------|
| 并发控制 | Redis SET NX + Lua释放 | 轻量高效，按用户维度加锁，锁超时自动释放 |
| 事务处理 | MySQL事务 + 手动连接管理 | 保证下单/平仓原子性，失败自动回滚 |
| 订单号生成 | 时间戳+随机码 | 有序可读，碰撞概率极低 |
| 定时任务 | setInterval | 轻量级，适合单进程部署 |
| 止损止盈触发 | floatingPnlUpdater统一检查 | 避免多处检查逻辑分散 |
| 费用计算 | 独立服务模块 | 复用性强，便于单元测试 |

### 5. 经验总结

1. 交易系统核心在于数据一致性：Redis锁+事务缺一不可
2. 费用计算引擎独立抽象后复用性极佳：下单、平仓、预估、结算都复用同一套
3. 定时任务职责要单一：止损止盈由floatingPnlUpdater统一检查，避免逻辑分散
4. 前端弹窗/表单功能不能留空壳：按钮有了必须有对应的完整交互
5. 文档必须与代码同步更新：开发完成后应立即更新验收文档
