### 3. 技术决策记录

| 决策项 | 选择 | 原因 |
|--------|------|------|
| 运营服务架构 | 统一 operationService | 6 大模块逻辑相关，统一维护方便 |
| 模拟盘费用展示 | 全品种合并 | 用户首次进入即可看到所有品种，标记配置来源 |
| 接口密钥安全 | 编辑前拉详情+空值不覆盖 | 列表脱敏不返回密钥，编辑时从详情回填 |
| Banner 跳转 | none/url/page/activity | 覆盖常见场景，用户端三种类型均实现 |
| 图片上传 | 复用已有上传接口 + URL 兜底 | 降低开发成本，兼容外部图床 |
| 响应结构 | 直接取 HTTP 封装返回值 | 项目 request 已解析为 res.data.data |

### 5. 经验总结

1. INSERT 操作在 mysql2 中返回 ResultSetHeader，不可按数组解构
2. 项目 HTTP 封装已统一解析 `res.data.data`，页面不应重复取 `.code`
3. 敏感数据（API 密钥）列表脱敏 + 编辑拉详情是标准做法
4. 全品种合并展示比仅显示已配置项更友好
