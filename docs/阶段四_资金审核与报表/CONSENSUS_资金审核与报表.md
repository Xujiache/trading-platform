# CONSENSUS - 阶段四：资金审核、收款账户、流水、报表

## 1. 明确的需求描述

本阶段实现完整资金管理体系（入金/出金/收款账户/流水）和报表系统（用户端3种+后台3种），支持人工审核模式、二维码上传、手续费估算、日期筛选。

- [x] 出金记录查询
- [x] 出金手续费估算

## 3. 技术实现方案

- 资金服务: `services/fundService.js` — 入金/出金/收款账户/流水/审核全流程
- 报表服务: `services/reportService.js` — 6种报表数据聚合
- 用户端路由: `routes/mobile/fund.js` + `routes/mobile/report.js`
- 后台路由: `routes/admin/fund.js` + `routes/admin/report.js`
- 文件上传: multer存储到uploads/qrcode，用户端有独立上传接口
- 支付配置: 从system_config表动态读取

## 4. 技术约束与集成方案

- 入金/出金为人工审核模式，非自动回调
- 出金仅支持实盘账户
- 资金操作需保证事务一致性

## 5. 任务边界限制

- 本阶段不实现自动支付回调
- 风控报表数据来源依赖阶段五的风控模块（可先用模拟数据占位）
