## 3. 技术实现方案

已确认的技术选型：

| 技术 | 选择 | 用途 |
|------|------|------|
| 后端框架 | Express | RESTful API |
| 数据库驱动 | mysql2/promise | 连接池 + async/await |
| 缓存 | ioredis | 验证码存储/限流 |
| 认证 | jsonwebtoken | 双token机制 |
| 密码 | bcryptjs | 哈希加密 |
| 文件上传 | multer | 磁盘存储 |
| 参数校验 | express-validator | 路由级校验 |
| 邮件 | nodemailer | SMTP发信 |
| 用户端 | UniApp x + Vue3 + UTS | 跨端 |
| 管理后台 | Art Design Pro (Vue3 + TS + Vite + Element Plus) | 后台UI |

## 4. 技术约束与集成方案

- 后端统一 RESTful API 风格
- 统一响应格式：`{ code, msg, data }`
- 统一错误码：200成功/400参数错/401未授权/403无权限/404不存在/500服务错误
- 后台前端基于 Art Design Pro 模板体系
- 用户端前端使用 UniApp x（.uvue + .uts）

## 5. 任务边界限制

- 本阶段不涉及任何交易、资金、行情业务逻辑
- 本阶段不涉及 WebSocket
- 用户表已预留 KYC/风险测评等字段，但不实现业务逻辑
