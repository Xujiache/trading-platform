# 黄金白银石油交易系统 - 项目说明文档

## 一、项目规划

### 1.1 项目概述
本系统是面向贵金属与能源 CFD 场景的多端协同交易系统，核心交易标的包括黄金、白银、原油。系统覆盖行情、交易、资金、风控、报表、公告、帮助、工单、客服与后台运营管理。

### 1.2 技术栈
| 端 | 技术栈 |
|----|--------|
| 用户端 | UniApp + Vue3 + ECharts |
| 管理后台 | Vue3 + TypeScript + Vite + Element Plus + TailwindCSS（Art Design Pro） |
| 后端 | Express + MySQL2 + Redis + JWT + WebSocket |

## 二、实施方案

### 2.1 文档体系
每个阶段独立文件夹，包含标准工作流文档：

```
docs/
├── 阶段一_基础设施与鉴权配置/     （7份文档）
├── 阶段二_行情服务与品种管理/     （7份文档）
├── 阶段三_交易引擎与账户体系/     （7份文档）
├── 阶段四_资金审核与报表/         （7份文档）
├── 阶段五_KYC风控与客服系统/      （7份文档）
├── 阶段六_后台运营能力/           （7份文档）
└── 阶段七_监控导出与稳定性收尾/   （7份文档）
```

每个阶段包含：
- **ALIGNMENT** - 对齐文档（需求理解、边界确认）
- **CONSENSUS** - 共识文档（验收标准、技术方案）
- **DESIGN** - 设计文档（架构图、接口契约）
- **TASK** - 任务拆分（原子任务、依赖关系）
- **ACCEPTANCE** - 验收记录（完成情况、测试结果）
- **FINAL** - 项目总结（质量评估、经验总结）
- **TODO** - 待办事项（缺少配置、操作指引）

### 2.2 开发原则
- 必须复刻现有系统全部业务能力，不允许删减
- 以 PRD 为唯一基准，代码已有能力必须纳入范围
- 交易、资金、风控三大模块必须后端业务规则先行
- 每完成一项任务立即标记完成并更新文档
