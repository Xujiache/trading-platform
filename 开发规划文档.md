## 一、项目概述

**项目名称**: 寰球汇金交易所  
**项目类型**: 黄金/白银/原油 CFD 差价合约在线交易平台  
**三端架构**:

| 子系统 | 技术栈 | 终端用户 |
|--------|--------|----------|
| Node.js 后端 | Fastify + TypeScript + Prisma + Redis + Socket.IO | 为前端提供 API 和实时数据 |
| UniApp 用户端 | UniApp + Vue3 + Pinia + UnoCSS + ECharts | 个人投资者（iOS/Android/H5） |
| 管理后台 | Arco Design Pro Vue + Vue3 + TypeScript + UnoCSS | 管理员/风控/财务/运营 |

## 五、API 路由设计

### 5.1 路由前缀

```
/api/mobile/*     → 移动端 API（需 mobile JWT）
/api/admin/*      → 管理后台 API（需 admin JWT）
/ws               → Socket.IO（行情 /market，客服 /chat）
/api/docs         → Swagger API 文档
```

### 5.2 移动端 API 清单（13 个模块）

#### 认证模块 `/api/mobile/auth`
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /register | 注册（手机号+验证码+密码） |
| POST | /login | 登录（邮箱/手机号+密码） |
| POST | /send-code | 发送短信验证码 |
| POST | /forgot-password | 忘记密码（验证码+新密码） |
| POST | /refresh-token | 刷新 Token |

#### 用户模块 `/api/mobile/user`
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /profile | 获取个人信息 |
| PUT | /profile | 修改个人资料 |
| PUT | /password | 修改密码 |
| POST | /avatar | 上传头像 |
| POST | /kyc | 提交 KYC 认证 |
| GET | /kyc/status | 查询 KYC 状态 |

#### 2FA 模块 `/api/mobile/two-factor`
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /generate | 生成 TOTP 二维码 |
| POST | /enable | 开启 2FA |
| POST | /disable | 关闭 2FA |
| POST | /verify | 验证 TOTP 验证码 |

#### 行情模块 `/api/mobile/market`
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /symbols | 获取品种列表 |
| GET | /symbols/:id | 获取品种详情 |

## 六、核心业务规则（开发时必须严格遵守）

### 6.1 保证金计算（使用 decimal.js）
```
已用保证金 = (合约单位 × 手数 × 开仓价格) / 杠杆比例
可用保证金 = 账户净值 - 已用保证金
净值 = 余额 + 浮动盈亏
保证金比例 = (净值 / 已用保证金) × 100%
```

### 6.2 浮动盈亏计算
```
做多: 浮盈 = (当前价 - 开仓价) × 合约单位 × 手数
做空: 浮盈 = (开仓价 - 当前价) × 合约单位 × 手数
```

### 6.3 点差成本
```
固定点差成本 = 点差值 × 合约单位 × 手数
浮动点差 = max(spreadMin, min(spreadMax, 实时计算点差))
```

### 6.4 手续费
```
按手数: 手续费 = 每手手续费 × 手数
按金额: 手续费 = 合约单位 × 手数 × 成交价 × 费率
单边 = 仅开仓收取
双边 = 开仓+平仓各收一次
阶梯费率: 根据用户等级或月交易量取对应阶梯
```

### 6.5 隔夜费
```
隔夜费 = (合约单位 × 手数 × 当日结算价 × 隔夜费利率) / 360
结算时间: 每日凌晨 5:00 北京时间
周三收取 3 倍（覆盖周六日）
多单/空单利率分别配置（可正可负）
```

### 6.6 强平规则
```
保证金比例 ≤ 100% → 推送预警通知
保证金比例 ≤ 50%  → 系统自动强平
强平顺序: 按持仓亏损幅度从大到小
穿仓 → 平仓所有持仓
强平记录不可篡改
```

### 6.7 出入金规则
```
入金: 最低金额可配置，支付后自动/手动审核
出金: 必须 KYC + 绑定本人银行卡 + 不超过可用余额 + 大额人工审核
出金手续费: 固定/比例可配
```

### 6.8 交易品种合约规格
| 品种 | 代码 | 合约单位 | 最小手数 | 默认杠杆上限 | 交易时间 |
|------|------|----------|----------|-------------|----------|
| 伦敦金 | XAUUSD | 100 盎司 | 0.01 | 1:200 | 周一至周五 24h |
| 伦敦银 | XAGUSD | 5000 盎司 | 0.01 | 1:100 | 周一至周五 24h |
| 美原油 | WTIUSD | 1000 桶 | 0.01 | 1:100 | 周一至周五 6:00-次日 5:00 |

## 十、开发规范

### 10.1 代码规范
- 后端全部使用 TypeScript，严格模式 `strict: true`
- 所有金额计算使用 `decimal.js`，禁止 JavaScript 原生浮点数运算
- 所有 API 入参使用 Zod Schema 校验
- 统一响应格式：`{ code: 0, data: {...}, message: "success" }`
- 错误响应格式：`{ code: number, message: string, errors?: [...] }`
- 日志使用 Pino，禁止 `console.log`

### 10.2 安全规范
- 密码使用 Argon2 哈希，禁止明文存储
- JWT mobile/admin 使用不同密钥
- 所有管理端操作记录操作日志
- 敏感字段脱敏（银行卡号、身份证号）
- SQL 全部通过 Prisma，禁止拼接 SQL

### 10.3 命名规范
- 文件名：camelCase（`tradeEngine.ts`）
- 类名：PascalCase（`TradeEngine`）
- 函数/变量：camelCase（`createOrder`）
- 常量：UPPER_SNAKE_CASE（`MAX_LOTS`）
- 数据库表/字段：camelCase（Prisma 默认映射）
- API 路径：kebab-case（`/api/mobile/bank-cards`）
